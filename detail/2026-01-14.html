1. 🚀 Google发布Veo 3.1视频生成模型重大更新

标签： Google Veo 3.1 Gemini API

Google于2026年1月14日发布Veo 3.1版本更新，显著提升视频生成能力。新版本支持通过参考图生成高质量视频，新增原生9:16垂直画幅输出，满足短视频平台创作需求。专业用户可使用1080P与4K超清上采样功能，提升输出画质。此外，Veo 3.1集成AI内容检测机制，支持识别视频是否由Google AI生成，增强内容溯源与版权保护能力。该模型通过Gemini API开放给开发者，适用于广告、影视制作与社交媒体内容创作，标志着Google在生成式视频领域的技术领先地位进一步巩固。

2. 🚀 百川智能开源235B参数医疗大模型Baichuan-M3

标签： 百川智能 Baichuan-M3 Hugging Face

百川智能于1月14日正式开源Baichuan-M3-235B医疗大模型，具备原生端到端问诊能力，可主动追问患者症状并识别潜在健康风险。该模型在多项医疗评测中表现超越GPT-5.2，幻觉率低至3.5%，显著提升诊断可靠性。Baichuan-M3支持多轮对话与医学知识推理，适用于在线问诊、辅助诊断与医学教育场景。模型已在Hugging Face与GitHub开源，并提供技术博客详解其架构与训练方法，推动医疗AI开源生态发展。

- https://github.com/baichuan-inc/Baichuan-M3-235B

3. 🚀 Google发布MedGemma 1.5多模态医疗模型

标签： Google MedGemma 1.5 MedASR

Google发布新一代开放医疗模型MedGemma 1.5，基于Gemma 3架构强化多模态处理能力。该模型支持CT、MRI等3D医学影像及病理切片的深度分析，提升疾病检测与诊断精度。MedGemma 1.5还集成医学语音转文本功能（MedASR），实现语音病历自动转录。模型已在Hugging Face开源，适用于临床辅助决策、医学研究与远程医疗系统，推动AI在精准医疗中的落地应用。

7. 🔧 NVIDIA联合发布TTT-E2E端到端长上下文训练方案

标签： NVIDIA TTT-E2E GitHub

NVIDIA联合伯克利等机构发布TTT-E2E（Test-Time Training End-to-End）方案，通过在推理阶段实时压缩上下文信息，显著降低长文本处理延迟。在2M超长上下文任务中，该方案推理速度比传统架构快35倍，提升大模型在文档分析、法律检索与科研辅助等场景的实用性。相关代码已在GitHub开源。
