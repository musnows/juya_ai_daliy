1. 📈 OpenAI正式启动ChatGPT广告测试

标签： OpenAI ChatGPT

OpenAI于2026年2月10日正式启动ChatGPT广告测试，标志着其商业化策略的重要一步。根据官方公告，广告将以非侵入式形式嵌入ChatGPT对话界面，优先面向免费用户展示，旨在提升用户体验的同时探索可持续的商业模式。此次测试初期覆盖部分英语用户，广告内容将与用户对话主题相关，但不会基于个人数据进行定向投放，以符合隐私保护原则。此举可能为AI助手类产品开辟新的收入来源，同时引发关于AI服务中立性与数据使用的讨论。

2. 🔧 Sam Altman宣布Codex将保持免费用户可用

标签： OpenAI Codex Sam Altman

OpenAI首席执行官Sam Altman在社交媒体上明确表示，Codex模型将继续对免费用户开放使用。这一表态回应了社区对AI编程工具可能全面付费化的担忧。Codex作为支撑GitHub Copilot等产品的核心模型，其持续免费可用性有助于维持开发者生态的活跃度，并巩固OpenAI在AI编程辅助领域的领先地位。

3. 🚀 OpenRouter上线 stealth 模型 Aurora Alpha

标签： OpenRouter Aurora Alpha

OpenRouter平台近日低调上线了一款名为Aurora Alpha的 stealth 模型，未进行大规模宣传。该模型通过OpenRouter的统一API接口提供服务，支持多模态输入与高效推理。尽管官方未披露具体架构细节，但社区测试显示其在代码生成与逻辑推理任务中表现优异，可能基于混合专家（MoE）架构设计，具备高扩展性与低延迟特性。

4. 🔧 Cursor上线编程模型Composer 1.5

标签： Cursor Composer 1.5

AI编程工具Cursor正式发布Composer 1.5版本，该模型在代码补全、重构建议与上下文理解方面显著增强。新版本支持跨文件语义分析，能够根据项目结构智能生成函数与类定义，并集成实时调试建议功能。Composer 1.5采用增量训练策略，在保持低资源占用的同时提升响应速度，适用于大型代码库开发场景。

5. 🚀 OpenAI逐步向第三方开放GPT-5.3-Codex

标签： OpenAI GPT-5.3-Codex

OpenAI开始向第三方开发者逐步开放GPT-5.3-Codex模型的API访问权限。该模型专为代码生成与理解优化，具备更强的上下文窗口处理能力与多语言支持。此次开放采用邀请制，首批合作伙伴包括多家企业级开发平台。此举有望推动AI编程工具在垂直行业的深度集成，加速软件开发自动化进程。

6. 🚀 美团AI编程工具CatPaw全量开放

标签： 美团 CatPaw

美团宣布其自研AI编程工具CatPaw正式向所有开发者全量开放。该工具基于美团内部大规模代码库训练，支持Java、Python、Go等多种语言，提供智能补全、Bug检测与文档生成功能。CatPaw采用本地化部署与云端协同架构，兼顾企业数据安全与使用便捷性，已在美团内部提升30%以上开发效率。

7. 🔧 Hugging Face发布Transformers.js v4预览版

标签： Hugging Face Transformers.js

Hugging Face推出Transformers.js v4预览版，支持在浏览器端直接运行先进Transformer模型。新版本优化了模型加载机制，引入WebGPU加速支持，显著提升推理性能。v4版本新增对视觉-语言模型的原生支持，并简化了API调用流程，使前端开发者能够更便捷地集成AI能力至Web应用中。

8. 🚀 魔搭ModelScope推出OpenAPI

标签： 魔搭 ModelScope

阿里云旗下魔搭开源社区ModelScope正式发布OpenAPI服务，提供统一接口调用平台内数百个开源模型。开发者可通过标准化RESTful API访问包括通义千问、视觉理解、语音合成等在内的多模态模型，支持按需计费与私有化部署选项。该举措旨在降低AI模型使用门槛，推动开源模型在企业场景中的落地应用。

9. 🔧 Sora上线Extensions续写功能

标签： Sora OpenAI

OpenAI视频生成模型Sora新增Extensions续写功能，允许用户基于已有视频片段进行内容扩展与风格延续。该功能通过时间轴对齐与语义一致性控制，实现流畅的视频段落衔接。用户可通过文本提示指导续写方向，支持多轮迭代编辑。此更新进一步拓展了Sora在创意内容生产中的实用性。

10. 🔧 智谱GLM-5架构细节曝光，总参数或达700B

标签： 智谱AI GLM-5

智谱AI的下一代大模型GLM-5架构细节在GitHub相关项目中被披露，显示其总参数量可能达到7000亿。该模型采用混合稀疏激活架构，结合MoE（Mixture of Experts）与稠密层设计，在保持高效推理的同时提升模型容量。社区分析指出，GLM-5在长文本理解与多轮对话任务中表现突出，预计将用于智谱清言等产品的升级。
