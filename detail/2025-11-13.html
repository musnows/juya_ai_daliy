1. 🚀 OpenAI发布GPT-5.1系列模型

标签： OpenAI GPT-5.1 System Card

OpenAI于近日正式发布GPT-5.1系列模型，作为GPT-5的增量升级版本，该模型在推理能力、多轮对话一致性、长上下文理解等方面进行了优化。根据官方发布的系统卡补充文档（System Card Addendum），GPT-5.1在安全对齐、幻觉抑制和指令遵循方面引入了新的微调策略，并增强了对复杂任务（如代码生成、数学推理）的支持。模型支持高达128K的上下文窗口，并在多模态输入（文本+图像）处理上表现更优。此次发布包含多个子版本（如5.1-turbo、5.1-standard），适用于不同性能与成本需求的部署场景。GPT-5.1的推出进一步巩固了OpenAI在通用大模型领域的领先地位，预计将在企业自动化、教育、内容创作等领域产生广泛影响。

- https://openai.com/index/gpt-5-system-card-addendum-gpt-5-1/

2. 🔧 谷歌新模型Riftrunner现身LMArena

标签： Google Riftrunner LMArena

谷歌尚未正式命名的新模型“Riftrunner”近日出现在LMArena（Large Model Arena）的匿名评测榜单中，引发业界关注。该模型在多项基准测试中表现接近或超越Gemini 1.5 Pro，尤其在代码生成、逻辑推理和长文本摘要任务中得分突出。LMArena采用盲测机制，用户无法识别模型来源，因此Riftrunner的真实身份尚未由谷歌官方确认，但测试目录（TestingCatalog）通过X平台披露其存在。该模型可能为谷歌下一代轻量化高性能模型，采用混合专家（MoE）架构，具备高推理效率。若属实，Riftrunner或将成为谷歌在边缘计算与实时应用部署中的重要工具。

3. 🚀 World Labs发布多模态世界模型Marble

标签： World Labs Marble 世界模型

World Labs正式发布其首款多模态世界模型Marble，该模型专注于构建可交互的3D环境模拟，支持从文本、图像或视频输入生成动态、可导航的虚拟世界。Marble具备物理引擎集成能力，可模拟光照、重力、碰撞等基础物理行为，并支持用户通过自然语言指令与场景中的对象进行交互。模型采用分层生成架构，先构建场景布局，再填充细节，适用于游戏开发、虚拟培训、数字孪生等场景。World Labs强调Marble的‘可编辑性’，允许开发者实时修改环境参数。该模型的发布标志着世界模型从静态生成向动态交互的重要演进。

🔗 相关链接：
- https://www.worldlabs.ai/blog/marble-world-model

4. 🚀 微博WeiboAI发布VibeThinker-1.5B模型

标签： WeiboAI VibeThinker-1.5B Hugging Face

5. 🚀 LMArena发布Code Arena编程评测平台

标签： LMArena Code Arena 编程评测

LMArena团队正式推出Code Arena，一个专注于编程能力评估的匿名模型评测平台。该平台采用与LMArena相同的盲测机制，用户提交编程问题（如算法实现、代码修复、调试建议），由不同大模型生成答案，系统自动评分并匿名展示结果。Code Arena支持Python、JavaScript、Java等主流语言，涵盖LeetCode风格题目、真实项目Bug修复、API集成等场景。评测维度包括代码正确性、可读性、效率与安全性。平台旨在为开发者提供客观的模型编程能力对比，推动AI编程助手的技术进步，并可能成为GitHub Copilot、Cursor等工具的性能参考标准。

6. 🚀 TRAE发布3.0版本并上线TRAE SOLO

标签： TRAE TRAE SOLO TRAE 3.0

TRAE AI发布其AI开发平台TRAE 3.0，并同步上线独立部署版本TRAE SOLO，限时免费三天。TRAE 3.0引入增强的代码生成引擎，支持从自然语言描述自动生成完整项目结构，包括前端、后端、数据库配置与API文档。TRAE SOLO允许企业将平台私有化部署于本地服务器，满足数据安全与合规需求。新版本支持多模型切换（如接入GPT、Claude、Qwen等），并集成CI/CD流水线自动化功能。限时免费策略旨在吸引开发者试用，推动平台生态建设。TRAE 3.0的发布标志着AI驱动软件开发向企业级应用迈出关键一步。

7. 🔧 GitHub Copilot支持Agent特定指令

标签： GitHub Copilot Agent指令 代码审查

GitHub Copilot正式支持Agent特定指令（Agent-Specific Instructions），允许开发者为Copilot的编码代理（Coding Agent）和代码审查代理（Code Review Agent）配置自定义行为规则。用户可通过配置文件（如.copilot-instructions）指定代理在生成代码时应遵循的编码规范、安全策略、框架偏好或注释风格。例如，可要求代理避免使用特定API、优先使用函数式编程范式或自动添加类型注解。该功能显著提升了Copilot在团队协作和大型项目中的可控性与一致性，减少人工审查负担。支持指令的代理已在GitHub Copilot Enterprise中上线，进一步巩固其作为AI编程助手的领先地位。

🔗 相关链接：
- https://github.blog/changelog/2025-11-12-copilot-code-review-and-coding-agent-now-support-agent-specific-instructions/

9. 🔧 Google发布AI Agents系列白皮书

标签： Google AI Agents MCP

Google通过Kaggle平台发布三份关于AI Agents的技术白皮书，系统阐述其Agent架构设计、工具集成与上下文管理机制。首份《Introduction to Agents》定义Agent为具备目标导向、自主决策与工具调用能力的AI系统；第二份《Agent Tools and Interoperability with MCP》介绍模型上下文协议（MCP），推动不同Agent间工具互操作；第三份《Context Engineering, Sessions and Memory》提出上下文工程框架，支持会话记忆、长期记忆与记忆压缩技术。白皮书强调Agent在自动化工作流、智能客服、科研辅助中的潜力，并为开发者提供设计指南。此举表明Google正构建统一的AI Agent技术标准体系。

- https://www.kaggle.com/whitepaper-agent-tools-and-interoperability-with-mcp
- https://www.kaggle.com/whitepaper-context-engineering-sessions-and-memory

13. 🚀 Hugging Face发布FinePDF-edu数据集
