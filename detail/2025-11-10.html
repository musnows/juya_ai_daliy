1. 🔧 xAI API上线MCP与远程工具支持

标签： xAI MCP API

xAI于近日正式在其API平台上线了对MCP（Model Control Protocol）协议的支持，并开放远程工具调用能力。此举允许开发者通过标准化的MCP协议与xAI大模型进行深度集成，实现跨系统、跨平台的自动化交互。新增的远程工具支持意味着外部应用可通过API安全调用本地或云端工具，提升AI代理在复杂任务中的执行能力。该更新标志着xAI正加速构建以模型为中心的开发生态，推动AI代理在真实业务场景中的落地。目前该功能已在官方API文档中提供接入指南，支持主流编程语言SDK。

2. 🚀 TRAE国际版将发布TRAE SOLO

标签： TRAE TRAE SOLO 端侧AI

TRAE AI官方宣布，其面向国际市场的产品线TRAE（国际版）即将正式发布独立版本——TRAE SOLO。该版本定位为轻量化、高隐私保护的本地化AI助手，支持离线运行与端侧推理，适用于对数据主权和响应延迟敏感的用户场景。TRAE SOLO将集成多模态理解与生成能力，并优化在消费级设备上的资源占用。此次发布标志着TRAE从云端协同架构向端侧智能延伸，进一步拓展其在个人AI助手市场的布局。具体发布日期尚未公布，但官方已在X平台发布预告推文。

🔗 相关链接：
- https://x.com/Trae_ai/status/1987611151572889690

3. 📈 Google Nano-Banana 2现身第三方平台

标签： Google Nano-Banana 2 轻量化模型

代号为'Google Nano-Banana 2'的疑似谷歌内部AI实验项目近日被曝光于第三方技术社区平台。该名称首次出现在X用户@ai_for_success的推文中，并附有相关技术截图与性能参数片段，显示其可能为轻量化多模态模型，具备图像描述、语音识别与文本生成能力，模型参数规模约为2B，专为移动端与边缘设备优化。尽管谷歌尚未正式确认该项目，但'Nanobanana'系列此前已被证实为谷歌内部用于探索高效AI部署路径的原型系统。此次Nano-Banana 2的泄露可能预示谷歌将在端侧AI领域推出新一代轻量模型。

4. 🔧 MoonshotAI开源kosong框架

标签： MoonshotAI kosong 长上下文建模

MoonshotAI正式开源其内部研发的AI开发框架——kosong，项目已上传至GitHub（github.com/MoonshotAI/kosong）并发布官方文档网站（moonshotai.github.io/kosong）。kosong是一个专注于长上下文建模与高效推理的轻量级框架，支持动态上下文压缩、注意力机制优化与多轮对话状态管理，特别适用于处理超长文本输入的生成式AI应用。该框架采用Python编写，兼容主流深度学习库，并提供预训练模型集成接口。开源kosong旨在降低长文本AI应用的开发门槛，推动社区在RAG、文档问答、代码生成等领域的创新。MoonshotAI表示将持续维护并收集社区反馈。

🔗 相关链接：
- https://github.com/MoonshotAI/kosong
- https://moonshotai.github.io/kosong/

5. 🔧 Firefox浏览器默认启用LLM功能

标签： Firefox LLM Mozilla

Mozilla Firefox浏览器在最新版本中开始默认启用内置的LLM（大语言模型）功能，引发用户与开发者社区关注。据技术博客equk.co.uk报道，Firefox通过集成本地运行的轻量级语言模型，支持页面内容摘要、智能搜索建议与隐私保护型AI交互。该功能依赖本地计算资源，不强制上传用户数据，但默认开启引发部分用户对隐私与性能影响的担忧。GitHub用户equk已发布工具ffox_profile_tools，允许用户手动禁用或配置相关功能。Mozilla此举标志着主流浏览器正加速向AI原生方向演进，Firefox试图在隐私与智能体验之间寻求平衡。

- https://github.com/equk/ffox_profile_tools
