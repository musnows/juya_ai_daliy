1. 🚀 OpenAI 推出 Codex 用例库，展示应用场景与使用技巧

标签： OpenAI Codex

OpenAI 正式推出 Codex 用例库，旨在为开发者提供丰富的应用场景示例和实用技巧，帮助更好地理解和应用 Codex 模型。该用例库涵盖多种编程任务、自动化脚本生成及代码补全等典型使用场景，通过实际案例降低技术门槛，提升开发效率。此举有助于推动 Codex 在企业级开发、教育及自动化工具中的普及，进一步巩固 OpenAI 在 AI 编程助手领域的领先地位。

2. 🔧 飞书开源 lark-cli，支持 AI Agent 覆盖11个业务域

标签： 飞书 lark-cli AI Agent

飞书宣布开源命令行工具 lark-cli，该工具集成 AI Agent 能力，支持跨 11 个业务域的自动化操作，包括日程管理、文档处理、审批流程等。lark-cli 的开源将促进企业内部工具链的智能化升级，提升办公自动化水平，同时为开发者提供可扩展的 AI 集成框架，推动飞书生态的技术开放与协作创新。

3. 📈 OpenAI 明确 Sora 应用4月停服，API将于9月下线

标签： OpenAI Sora

OpenAI 正式宣布，Sora 应用将于2026年4月停止服务，其 API 接口则计划于同年9月完全下线。此举标志着 Sora 作为独立产品的生命周期结束，可能与其技术路线调整或资源重新分配有关。用户需在截止日期前完成数据迁移或替代方案部署，相关开发者应关注 OpenAI 后续发布的替代产品或集成路径。

4. 🔧 fnOS 新增 OpenClaw 应用中心安装支持

标签： fnOS OpenClaw

fnOS 操作系统宣布新增对 OpenClaw 应用中心的安装支持，用户可通过官方渠道直接安装和管理 OpenClaw 平台上的应用。这一集成提升了 fnOS 的应用生态丰富度，增强了其在去中心化应用分发领域的竞争力，也为 OpenClaw 开发者提供了更广泛的用户触达渠道。

5. 📈 RaBitQ 团队质疑 Google TurboQuant 论文存在学术不端

标签： RaBitQ Google TurboQuant

RaBitQ 研究团队公开质疑 Google 发布的 TurboQuant 论文存在学术不端行为，指出其在实验数据、方法描述或结果复现方面可能存在瑕疵。该质疑引发学术界对大型科技公司研究成果透明度的关注，若属实，可能影响 TurboQuant 技术的可信度及其在模型量化领域的应用推广。

6. 📈 谷歌拟出资数十亿支持 Anthropic 得州数据中心建设

标签： Google Anthropic

据知情人士透露，谷歌计划出资数十亿美元，支持 Anthropic 在得克萨斯州建设大型数据中心。该投资不仅体现谷歌对 Anthropic 的战略重视，也反映其通过基础设施合作深化在生成式 AI 领域布局的意图。此举有望加速 Anthropic 模型训练与部署能力，同时增强谷歌云在 AI 算力生态中的影响力。

7. 🚀 Google 新一代开源模型 Gemma 4 或将发布

标签： Google Gemma 4

据多方消息，Google 即将发布新一代开源大模型 Gemma 4。作为 Gemma 系列的重大升级，Gemma 4 预计在模型规模、推理效率和多模态能力方面实现显著提升。其开源策略将继续吸引全球研究者和开发者社区参与，推动轻量化、可定制 AI 模型的发展，进一步巩固 Google 在开源 AI 领域的技术领导地位。
