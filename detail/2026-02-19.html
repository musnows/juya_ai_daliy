1. 🚀 Google DeepMind 发布 Lyria 3 音乐生成模型并集成至 Gemini

标签： Google DeepMind Lyria 3 Gemini

Google DeepMind 于2026年2月19日正式发布其最新一代音乐生成模型 Lyria 3，并将该模型集成至 Gemini 平台，用户可通过 Gemini 应用直接体验高质量音乐创作功能。Lyria 3 在音乐结构理解、多乐器编排和风格迁移方面实现显著提升，支持生成长达数分钟的连贯音乐作品，涵盖流行、古典、电子等多种风格。据官方博客介绍，该模型采用先进的扩散架构与大规模音乐数据集训练，具备更强的旋律一致性与情感表达能力。此次上线标志着 Google 在生成式音频领域的技术领先地位进一步巩固，也为创作者、内容制作人和普通用户提供了更强大的AI辅助工具。相关技术文档与访问入口已发布于 deepmind.google/models/lyria 及 blog.google 官方页面。

2. 🔧 xAI 推出 Grok 4.20 Heavy 支持16-Agent协作模式

标签： xAI Grok 4.20 Heavy

xAI 公司发布 Grok 4.20 Heavy 版本，首次引入16-Agent协作推理机制，显著提升复杂任务处理与多轮对话能力。该模式允许多个AI代理协同工作，分工处理信息检索、逻辑推理与内容生成等环节，适用于科研分析、代码审查等高阶应用场景。Elon Musk 在 X 平台确认此更新，强调其在提升响应质量与系统鲁棒性方面的突破。尽管具体技术细节未完全公开，但社区推测其基于强化学习与分布式推理架构优化。该版本标志着多智能体系统在大型语言模型中的实际应用迈出关键一步。

3. 🚀 Prime Intellect 开源106B参数 MoE 模型 INTELLECT-3.1

标签： Prime Intellect INTELLECT-3.1

AI 研究机构 Prime Intellect 正式开源其1060亿参数的混合专家模型（MoE）INTELLECT-3.1，模型权重与训练配置均可在 Hugging Face 平台获取。该模型采用稀疏激活架构，在保持高性能的同时降低推理成本，适用于大规模语言理解与生成任务。官方表示，INTELLECT-3.1 在多项基准测试中表现优于同规模密集模型，尤其在多语言与代码生成任务上具备优势。此举进一步推动开源大模型生态发展，为学术界与中小企业提供先进研发基础。

4. 🚀 印度AI企业 Sarvam 发布 Sarvam-30B 与 105B 大模型

标签： Sarvam AI Sarvam-30B Sarvam-105B

印度人工智能公司 Sarvam AI 宣布推出两款自研大语言模型 Sarvam-30B 与 Sarvam-105B，分别面向中等规模部署与高性能应用场景。据 TechCrunch 报道，这两款模型专注于印度本地语言支持与区域文化适配，具备强大的多语言处理能力，尤其在印地语、泰米尔语等南亚语种上表现突出。Sarvam 强调其对开源AI可行性的长期投入，计划逐步开放模型权重与训练数据。此次发布被视为印度在全球AI竞争中提升自主技术能力的重要一步。

5. 🔧 OpenAI 开源 Codex App Server 支持 ChatGPT 登录

标签： OpenAI Codex App Server

OpenAI 正式开源 Codex App Server，开发者可通过 GitHub 获取完整代码库（位于 codex-rs/app-server 目录），实现本地部署与自定义集成。该服务支持使用 ChatGPT 账户进行身份验证，便于构建私有化AI应用接口。此举降低了开发者接入口语化编程与智能代码辅助功能的门槛，同时增强数据控制权与安全性。官方文档指出，该服务器兼容主流IDE插件，可无缝嵌入现有开发流程，推动AI编程工具普及。

6. 🚀 Figma 推出 Claude Code to Figma 功能

标签： Figma Claude Code

设计协作平台 Figma 宣布集成 Anthropic 的 Claude Code 能力，推出“Code to Figma”功能，允许开发者将代码片段直接转换为可编辑的UI设计稿。该功能基于 Claude 的代码理解与视觉还原能力，支持 React、SwiftUI 等主流框架输出，大幅提升产品原型制作效率。Figma 官方博客表示，此功能已进入公测阶段，未来将扩展至设计系统同步与组件自动生成等高级场景，强化设计与开发之间的协作闭环。

7. 🔧 OpenAI 发布智能合约安全基准 EVMbench

标签： OpenAI EVMbench

OpenAI 推出专为以太坊虚拟机（EVM）智能合约设计的安全评估基准 EVMbench，旨在系统化检测合约漏洞与逻辑缺陷。该基准涵盖重入攻击、整数溢出、权限控制等常见风险类别，提供标准化测试用例与评分体系。研究人员可利用 EVMbench 训练和评估AI模型在区块链安全领域的表现，推动自动化审计工具发展。OpenAI 表示，此举是其拓展AI在关键基础设施安全中应用的战略组成部分。

8. 🔧 Gemini CLI 更新至 v0.29.0 实现每周迭代发布

标签： Gemini CLI

Google 的 Gemini 命令行工具（CLI）发布 v0.29.0 版本，并宣布采用每周更新机制，加速功能迭代与问题修复。新版本优化了API调用效率、错误提示清晰度及多账户切换体验。开发团队在 GitHub Discussions 中透露，未来将引入插件系统与本地缓存策略，进一步提升开发者工作流集成能力。该举措反映 Google 对开发者生态的重视，强化 Gemini 在终端环境下的实用性与响应速度。

9. 📈 谷歌、OpenAI 与微软宣布在印度扩大AI布局

标签： Google OpenAI Microsoft

谷歌、OpenAI 和微软联合宣布在印度启动多项AI发展计划，涵盖教育合作、技能培训与基础设施建设。谷歌通过 AI Impact Summit 2026 活动展示其在医疗、农业领域的本地化AI解决方案；OpenAI 聚焦高等教育，推动AI课程与实验室建设；微软则加强云服务与中小企业支持。三方均表示将投入资源培养本土AI人才，响应印度政府‘数字印度’战略。此举标志着全球科技巨头在亚洲新兴市场AI竞争进入深化阶段。

10. 📈 Perplexity 移除搜索平台广告以优化用户体验

标签： Perplexity

AI 搜索公司 Perplexity 宣布全面撤下其搜索平台上的广告内容，转向订阅制与增值服务盈利模式。据《金融时报》报道，此举旨在提升搜索结果的中立性与用户体验纯净度，避免商业利益干扰信息呈现。公司表示，未来将通过 Pro 会员计划提供无广告、优先访问新功能等权益。这一转变可能影响AI搜索行业的商业化路径，引发对可持续盈利模式的重新思考。

11. 📈 World Labs 完成融资并与 Autodesk 达成战略合作

标签： World Labs Autodesk

空间智能公司 World Labs 宣布完成新一轮融资，并正式与 Autodesk 建立合作伙伴关系，共同开发面向建筑与工程领域的3D场景理解与生成工具。World Labs 专注于将AI应用于物理世界建模，其技术可自动解析现实环境并生成高精度数字孪生。合作将整合 Autodesk 的 BIM 平台与 World Labs 的感知算法，提升设计自动化水平。融资细节未披露，但公司表示资金将用于团队扩张与产品商业化落地。

12. 🔧 智谱AI 发布 GLM-5 技术报告展示多模态能力突破

标签： 智谱AI GLM-5

中国AI公司智谱AI 发布 GLM-5 系列模型的技术报告（arXiv:2602.15763），详细阐述其在多模态理解、长上下文处理与推理一致性方面的创新。GLM-5 支持图像、文本、音频联合输入，具备跨模态对齐与生成能力，在科学问答与复杂指令遵循任务中表现优异。报告显示，模型在内部评估中较前代提升超过30%的综合性能。智谱AI 表示将持续推进开源与行业应用，强化中文AI生态建设。
