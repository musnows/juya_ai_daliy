1. 🚀 Google 发布 Lyria 3 Pro 和 Lyria 3 Clip 音乐生成模型

标签： Google Lyria 3 Pro Lyria 3 Clip

谷歌于2026年3月26日正式发布新一代音乐生成模型 Lyria 3 Pro 和 Lyria 3 Clip。其中，Lyria 3 Pro 支持最长生成三分钟的专业级音轨，具备高精度旋律、和声与节奏控制能力，适用于音乐创作、广告配乐及影视原声制作。Lyria 3 Clip 则专注于短片段音乐生成，优化实时交互体验，适合短视频平台与社交媒体内容创作。两款模型均基于谷歌自研的扩散与自回归混合架构，支持多乐器编排与风格迁移，用户可通过自然语言提示词精确控制输出效果。此次升级显著提升了生成音乐的连贯性与情感表达能力，标志着AI音乐生成技术向专业创作领域迈出关键一步。

2. 🔧 美团开源 LongCat-Next 多模态离散自回归模型

标签： 美团 LongCat-Next 龙猫团队

美团旗下龙猫团队正式开源 LongCat-Next，一款基于离散自回归架构的多模态大模型。该模型支持图像、文本与音频的联合理解与生成，采用 token 化表示实现跨模态对齐，具备高效的推理能力与较低的计算资源需求。LongCat-Next 在视觉问答、图文生成及多模态对话任务中表现优异，适用于智能客服、内容审核与多媒体内容生产等场景。开源版本包含基础版与微调版，支持开发者本地部署与二次开发，进一步推动多模态AI技术的普及与应用。

3. 🚀 北大与字节跳动联合发布开源视频生成模型 Helios

标签： 北京大学 字节跳动 Helios

北京大学与字节跳动联合研发并开源视频生成模型 Helios。该模型支持从文本描述生成高质量短视频片段，具备时序一致性控制与场景动态建模能力，可生成分辨率达1080p、时长达10秒的视频内容。Helios 采用分层扩散架构与运动感知模块，有效解决传统模型中常见的画面闪烁与逻辑断裂问题。开源版本提供预训练权重与训练代码，支持学术研究与商业应用，有望推动AI视频生成在影视制作、教育培训与数字人驱动等领域的落地。

4. 🔧 Google 明确 Gemini CLI 支持 ACP、A2A 及 Headless 模式

标签： Google Gemini CLI ACP

谷歌正式明确 Gemini CLI 的使用权限与功能范围，确认其支持 Agent Communication Protocol（ACP）、Application-to-Application（A2A）接口以及 Headless 无头模式运行。这一更新使开发者能够将 Gemini 集成至自动化工作流、后台服务与第三方系统中，提升AI代理的协同效率与部署灵活性。Gemini CLI 提供命令行工具与API接口，支持批量处理、任务调度与日志监控，适用于企业级AI应用开发与运维管理。

5. 📈 GitHub 更新 Copilot 政策，个人版默认用于模型训练

标签： GitHub Copilot

GitHub 宣布更新 Copilot 服务政策，明确个人版用户代码数据将默认用于AI模型训练，用户需手动选择退出。此举旨在提升代码补全与生成模型的准确性，但引发部分开发者对隐私与代码版权的担忧。企业版仍提供数据隔离选项，确保敏感代码不被用于训练。GitHub 表示将加强数据匿名化处理，并承诺不存储原始代码文件，仅使用抽象特征进行模型优化。

6. 🔧 Figma 向 AI agents 开放 Canvas 接口

标签： Figma Figma Canvas

Figma 宣布向AI代理开放其核心设计平台 Figma Canvas，允许第三方AI系统通过API直接读取、修改与生成设计稿。此举标志着设计工具向智能化协作迈出重要一步，AI代理可实现自动布局调整、组件生成与风格迁移等功能。开发者可利用该接口构建智能设计助手，提升UI/UX设计效率。Figma 同时推出沙箱环境与权限控制机制，确保设计数据安全与操作可追溯。

7. 🚀 谷歌推出 Vibe Coding XR，自然语言生成 WebXR 应用

标签： Google Vibe Coding XR WebXR

谷歌发布 Vibe Coding XR 工具，允许开发者通过自然语言描述直接生成 WebXR 应用，如虚拟现实（VR）与增强现实（AR）体验。该工具集成语义解析与3D场景构建引擎，支持快速原型开发与跨平台部署。用户可输入“创建一个带喷泉的虚拟花园”等指令，系统自动生成可交互的XR场景。Vibe Coding XR 降低了XR开发门槛，推动沉浸式内容在教育与娱乐领域的普及。

8. 🔧 Mozilla 推出 cq 项目实现 Agent 知识共享标准

标签： Mozilla cq

Mozilla 发布 cq 项目，旨在建立统一的AI代理知识共享标准。该项目定义了一套结构化数据格式与通信协议，使不同AI系统能够安全、高效地交换知识与经验。cq 支持知识溯源、权限控制与版本管理，适用于多代理协作、联邦学习与跨平台AI集成。Mozilla 呼吁行业共同采纳该标准，推动AI生态的开放与互操作性。

9. 🔧 Anthropic 为 Claude 移动端新增工作相关功能

10. 🔧 OpenClaw 发布新版，增强兼容性并支持 Teams 与 Slack

11. 🔧 Nous Research 发布 Hermes Agent v0.4.0

Nous Research 发布 Hermes Agent v0.4.0，重点优化多轮对话记忆与任务规划能力。该版本引入长期记忆存储机制与动态目标分解算法，使AI代理能在复杂任务中保持上下文一致性。Hermes Agent 支持插件扩展与自定义工作流，适用于自动化客服、数据分析与项目管理等场景。开源社区已贡献多个第三方插件，进一步丰富其应用生态。

MiniMax 开源 Office Skills 工具集，覆盖 Word、Excel、PowerPoint 与 PDF 四大主流办公文档格式的自动化处理能力。该工具集提供文档解析、内容提取、格式转换与批量操作API，支持与主流办公软件无缝集成。开发者可利用其构建智能文档助手，实现报告生成、数据汇总与格式标准化等任务，显著提升办公自动化水平。

14. 🔧 Google Research 发布 TurboQuant，KV 缓存降低6倍

标签： Google Research TurboQuant

Google Research 发布 TurboQuant 技术，通过新型量化算法将大模型推理过程中的键值（KV）缓存占用降低至原来的1/6。该技术采用动态精度调整与稀疏表示策略，在保持模型性能的同时显著减少内存消耗与延迟。TurboQuant 适用于资源受限的边缘设备与大规模并发服务，有望推动大模型在移动端与实时系统中的部署。

OpenAI 公开 Model Spec 的制定逻辑与配套评估套件，详细说明其AI模型行为准则的设计原则与测试方法。Model Spec 涵盖安全性、公平性、透明度与用户控制等方面，通过量化指标与场景测试确保模型合规。评估套件开源供第三方使用，支持对各类AI系统进行行为审计。此举增强公众对AI治理的信任，推动行业建立统一伦理标准。

16. 📈 ARC 发布 ARC-AGI-3 基准，现有模型得分均不足1%

ARC 发布新一代通用人工智能基准 ARC-AGI-3，测试结果显示当前所有主流AI模型在该基准上的得分均低于1%。ARC-AGI-3 包含更复杂的推理、抽象与跨领域迁移任务，强调真实世界问题解决能力。该结果凸显当前AI系统在通用智能方面仍存在巨大差距，为未来研究方向提供重要参考。ARC 呼吁加强基础认知架构与元学习能力的探索。

17. 📈 OpenAI 节省算力研发下代模型，Sam Altman 转攻数据中心建设

标签： OpenAI Sam Altman

18. 📈 OpenAI 推出安全漏洞赏金计划，聚焦AI滥用风险

标签： OpenAI
